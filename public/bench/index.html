React vs SolidJS
How it works: Service workers and browser caches are nuked before each run. Each framework loads in a cache-busted iframe, opens 15 windows, then runs drag, resize, snap, focus cycling, minimize/restore, expo/scale views, tab reorder, and batch close. React runs first, then SolidJS, under identical conditions. Per-resource cache hit/miss is tracked via PerformanceResourceTiming.transferSize.
